Anti-Prompt Injection

Scans untrusted content (web search results, scraped pages, image descriptions, skill outputs) for prompt injection attempts. Suspicious content is wrapped in safety markers so the AI model knows it's untrusted data.

Tool Guardrails

Require explicit user approval before executing shell commands. When disabled, the AI can run shell commands without asking for permission first (allowlist/blocklist/dangerous pattern checks still apply).

Security Info

Anti-Injection protects against malicious instructions hidden in web pages, search results, images, and skill outputs. It uses pattern matching to detect common injection techniques like "ignore previous instructions" or role impersonation.

Tool Guardrails add a confirmation step before the AI executes shell commands. Even when disabled, the built-in blocklist (rm -rf, dd, mkfs, etc.) and dangerous pattern detection remain active.

Both features are enabled by default. Disable only if you understand the risks.
