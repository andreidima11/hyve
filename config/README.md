# Local configuration (not in git)

| File | Purpose |
|------|---------|
| `device_aliases.yaml` | Friendly device names per integration (created at runtime) |

Copy `device_aliases.example.yaml` to `device_aliases.yaml` if you need a
starter file manually. Hyve creates an empty aliases file when you rename a
device in the UI.

Global settings live in `config.json` (repo root, also gitignored).
