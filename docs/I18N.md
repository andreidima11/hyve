# Traduceri (i18n) în Hyve

Hyve folosește **traduceri decentralizate**: fiecare modul (integrare, add-on, feature platformă) își păstrează stringurile în folderul propriu. Fișierele mamă `static/js/lang/en.js` și `ro.js` conțin doar **shell-ul UI** (login, navigare, setări globale, dashboard generic).

**Regula de aur:** dacă stringul aparține unei integrări, unui add-on sau unui modul platformă dedicat (camere, scene, backup…) → **nu** îl pui în `en.js` / `ro.js`.

---

## Hartă rapidă

| Ce traduci | Unde pui fișierele | Namespace în UI (`t('…')`) |
|------------|-------------------|----------------------------|
| Shell UI global | `static/js/lang/en.js`, `ro.js` | `login.*`, `config.*`, `dashboard.*`, … |
| Integrare (config, senzori, catalog) | `components/<slug>/translations/{en,ro}.json` | `components.<slug>.*` |
| Integrare custom (user) | `custom_components/<slug>/translations/` | la fel |
| Descriere în catalog integrări | cheie `catalog_desc` în JSON-ul integrării | `components.<slug>.catalog_desc` |
| Add-on — UI partajat (instalare, loguri) | `addons/translations/{en,ro}.json` | `apps.*` |
| Add-on — stringuri specifice slug | `addons/available/<slug>/translations/` | `addons.<slug>.*` |
| Feature platformă (camere, scene, …) | `core/i18n/<bundle>/translations/{en,ro}.json` | `<bundle>.*` (ex. `cameras.not_found`) |
| Erori API (backend) | aceleași bundle-uri + `error_detail("cheie", params)` | traduse în browser cu `translateApiDetail()` |

Limbi suportate azi: **en**, **ro**.

---

## Cum ajung stringurile în browser

```
static/js/lang/en.js + ro.js     ← încărcate la initI18n() (shell)
        │
        ▼
GET /api/i18n/bundles?lang=…     ← după autentificare + la schimbarea limbii
        │
        ├── core/i18n/*/translations/     → cameras.*, scenes.*, updates.*, …
        ├── components/*/translations/    → components.<slug>.*
        └── addons/translations/ + per-slug → apps.*, addons.<slug>.*
        │
        ▼
mergeBundledTranslations() în static/js/lang/index.js
        │
        ▼
t('cameras.not_found', { entity_id: '…' })
```

**Cod relevant:**

| Rol | Fișier |
|-----|--------|
| Merge + `t()` | `static/js/lang/index.js` |
| Încărcare la boot | `static/js/boot/state.ts` (+ `bindings/dom_ready.ts`) → `loadBundledTranslations()` via `initI18n()` |
| Schimbare limbă | `static/js/config/ui_language.ts` |
| API | `routers/i18n.py` → `GET /api/i18n/bundles` |
| Agregare backend | `core/i18n/bundles.py` |
| Bundle platformă | `core/i18n/platform.py` |
| Bundle integrări | `integrations/component_i18n.py` |
| Bundle add-on-uri | `addons/i18n.py` |
| Warm cache la startup | `core/http/startup_phases.py` |

Endpoint legacy `GET /api/i18n/components` returnează doar slice-ul `components.*` pentru integrările **configurate** — preferă `bundles` pentru UI.

## Format fișiere JSON (integrări / add-on-uri)

`components/my_service/translations/en.json`:

```json
{
  "config": {
    "title": "My Service account"
  },
  "catalog_desc": "Short description shown in the integrations catalog"
}
```

În UI:

- `t('components.my_service.config.title')`
- `t('components.my_service.catalog_desc')` — folosit automat dacă `ui_catalog.json` nu setează `description_key` (default: `components.<slug>.catalog_desc` în `core/ui_catalog.py`).

**Fallback:** dacă lipsește `ro.json`, se folosește `en.json` din același folder.

---

## Format bundle platformă

`core/i18n/cameras/translations/en.json`:

```json
{
  "not_found": "Camera {entity_id} not found.",
  "snapshot_timeout": "Snapshot timed out."
}
```

Namespace-ul top-level este **numele folderului** (`cameras`, `updates`, `backup`, `scenes`, `integrations`, `hy`).

Adaugă un bundle nou doar pentru feature-uri transversale care nu aparțin unei singure integrări.

---

## Erori API (backend)

Nu returna texte hardcodate în română sau engleză din routere.

```python
from core.http.errors import error_detail
from fastapi import HTTPException

raise HTTPException(404, error_detail("cameras.not_found", {"entity_id": entity_id}))
```

Răspuns: `{ "key": "cameras.not_found", "params": { "entity_id": "…" } }`.

Frontend: `translateApiDetail(detail)` din `static/js/lang/index.js`.

Cheia trebuie să existe în bundle-ul potrivit (`core/i18n/<bundle>/translations/` sau integrare).

---

## Notificări și payload-uri structurate

Notificările push/SSE pot trimite `title_key` / `body_key` (+ `params`) în loc de text brut — UI le traduce la afișare (`static/js/notifications.ts`).

Aceeași regulă: cheile trăiesc în bundle-ul modulului, nu în `en.js`.

---

## Catalog integrări (`ui_catalog.json`)

| Câmp | Rol |
|------|-----|
| `title_key` | Titlu în listă (ex. `integrations.catalog.tapo` sau `components.tapo.config.title`) |
| `description_key` | Descriere; default server: `components.<slug>.catalog_desc` |

Stringurile pentru `title_key` / `description_key` trebuie să existe în bundle-ul indicat — de obicei `components/<slug>/translations/`, nu în mother lang.

Vezi și [CARDS_AND_INTEGRATIONS.md](CARDS_AND_INTEGRATIONS.md).

## Workflow: integrare nouă

1. Creează `components/<slug>/translations/en.json` și `ro.json`.
2. Folosește `t('components.<slug>.…')` în TS/JS (sau lasă `CONFIG_SCHEMA` + UI generic să citească chei declarative).
3. Adaugă `catalog_desc` pentru descrierea din catalog.
4. Erori API: `error_detail("components.<slug>.…")` sau chei din `core/i18n/<bundle>/` dacă e logică platformă (ex. camere).
5. Rulează auditul: `python scripts/check_i18n_keys.py`.
6. **Nu** modifica `static/js/lang/en.js` pentru stringuri de integrare.

---

## Workflow: add-on nou

1. Stringuri comune instalare/loguri: deja în `addons/translations/` (`apps.*`).
2. Stringuri specifice: `addons/available/<slug>/translations/{en,ro}.json` → `addons.<slug>.*`.
3. Custom add-on: `custom_addons/<slug>/translations/`.
4. Comportament special (detectare instalare pe disk, sync config, hints UI): `addons/available/<slug>/lifecycle.py` + `"lifecycle_module": "lifecycle"` în `manifest.json`. Dispatcher: `addons/lifecycle.py`. Teste: `tests/test_addon_lifecycle.py`.

Hook-uri opționale în `lifecycle.py`:

| Hook | Rol |
|------|-----|
| `detect_on_disk_version(manifest, project_root=…)` | Artefacte locale → versiune instalată |
| `after_config_update(config, manifest=…, slug=…)` | După salvare config în registry |
| `enrich_catalog_entry(entry, manifest=…, slug=…)` | Câmpuri extra pentru UI catalog |

---

## Workflow: limbă nouă (viitor)

1. `static/js/lang/<code>.js` — shell (copiază structura din `en.js`).
2. Înregistrează în `static/js/lang/index.js` (`LANGUAGES`, `AVAILABLE_LANGUAGES`).
3. Adaugă `<code>.json` în fiecare folder `translations/` (integrări, add-on-uri, `core/i18n/*/translations/`).
4. Extinde `SUPPORTED_LANGS` în `core/i18n/util.py`.

---

## Unelte

| Comandă | Scop |
|---------|------|
| `python scripts/check_i18n_keys.py` | Verifică că toate `t('…')` din `static/js/**` au cheie în mother lang **sau** bundle-uri decentralizate |
| `node scripts/extract_i18n_bundles.mjs` | Mută secțiuni greșit lăsate în `en.js`/`ro.js` în JSON-uri per folder (one-off / curățare) |
| `pytest tests/test_i18n_audit.py` | Rulează auditul în CI |
| `pytest tests/test_bundled_i18n.py` | Merge platformă + componente + add-on-uri |

---

## Anti-pattern-uri

| Nu face | Fă în schimb |
|---------|----------------|
| String integrare în `static/js/lang/en.js` | `components/<slug>/translations/` |
| `HTTPException(400, "Mesaj în română")` | `error_detail("cheie", params)` |
| `integrations.catalog.<slug>_desc` în mother lang | `catalog_desc` în folderul integrării |
| Editare `features.js` per vendor pentru texte | `CONFIG_SCHEMA` + chei i18n în bundle |
| Presupunere că `/api/i18n/components` e suficient | `loadBundledTranslations()` → `/api/i18n/bundles` |

---

## Backend UI strings (separat)

`core/backend_i18n.py` + `locales/{en,ro}.json` — texte pentru procese Python care nu trec prin browser (ex. brain). **Nu** se amestecă cu bundle-urile UI din `core/i18n/`.
